## Deploy to production

```bash
git checkout main
git merge dev
git push origin main
```

Both **Vercel** (frontend) and **Render** (backend) watch the `main` branch and redeploy automatically.

The Render start command runs DB migrations before booting:

```
drizzle-kit migrate && node dist/main.js
```

## Services

| Service | What it hosts | URL |
|---------|--------------|-----|
| Vercel | React SPA | Auto-assigned `.vercel.app` domain |
| Render | NestJS API | `wordchord-api.onrender.com` |
| Neon | PostgreSQL | Managed cloud Postgres |
| Grafana Cloud | Logs (Loki) | `grafana.com` |

## Environment variables

### Vercel (frontend)

```
VITE_API_URL=https://wordchord-api.onrender.com/api
```

### Render (backend)

```
DATABASE_URL=postgresql://...@neon.tech/...
JWT_SECRET=<openssl rand -hex 32>
ALLOWED_ORIGIN=https://your-app.vercel.app
PORT=10000
NODE_ENV=production
GOOGLE_CLIENT_ID=...
RESEND_API_KEY=...
APP_URL=https://your-app.vercel.app
LOKI_URL=https://<userId>:<apiKey>@logs-prod-XXX.grafana.net
```

## CI

GitHub Actions runs on every PR to `main`:

1. **Server** — Jest tests → NestJS build
2. **Client** — Vite build

See `.github/workflows/ci.yml`.
