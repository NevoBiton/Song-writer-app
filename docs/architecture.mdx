## Overview

```
Browser (React SPA)
      │  HTTPS + Bearer JWT
      ▼
NestJS API (Render)
      │
      ├── Drizzle ORM
      │       │
      │       ▼
      │   PostgreSQL (Neon)
      │
      ├── Google OAuth  (google-auth-library)
      ├── Password reset emails  (Resend)
      └── Logs → Grafana Loki
```

## Frontend

React 18 SPA hosted on **Vercel**. Key libraries:

- **React Router v6** — client-side routing
- **TanStack Query** — server state, caching, and mutations
- **Axios** — HTTP client with JWT interceptor
- **shadcn/ui + Tailwind CSS** — component library and styling
- **Sonner** — toast notifications

Auth tokens are stored in `localStorage` (remember me) or `sessionStorage` (session only). A response interceptor clears the token and redirects to `/sign-in` on any 401 from a protected endpoint.

## Backend

NestJS 10 API hosted on **Render**. Structure:

```
src/
├── main.ts              # Bootstrap: Helmet, CORS, ValidationPipe, Winston logger
├── app.module.ts        # Root module
├── logger/              # Winston + Grafana Loki transport
├── database/            # Global Drizzle ORM provider
├── auth/                # JWT strategy, Google OAuth, password reset
├── songs/               # Song CRUD + soft-delete + restore
└── users/               # Profile + password management
```

All business logic lives in **services**. Controllers are thin HTTP layers that validate DTOs and delegate to services.

Rate limiting: **100 requests / minute** per IP via `@nestjs/throttler`, applied globally.

## Database

PostgreSQL on **Neon** (serverless). Schema managed by **Drizzle ORM**.

| Table | Description |
|-------|-------------|
| `users` | Accounts (email unique, password hash nullable for Google users) |
| `songs` | Active songs with sections stored as JSON |
| `deleted_songs` | Soft-deleted songs (auto-purged after 30 days) |
| `password_reset_tokens` | Single-use tokens with 1-hour expiry |

Migrations live in `server/drizzle/`. Generate a new one after schema changes:

```bash
cd server && npm run db:generate
```

## Auth

1. User registers or signs in — server returns a signed **7-day JWT**
2. Frontend stores the JWT and attaches it as `Authorization: Bearer <token>` on every request
3. `JwtAuthGuard` (Passport) validates the token on protected routes
4. Google sign-in: frontend sends the Google ID token → server verifies with `google-auth-library` → upserts user → returns JWT

## Observability

Logging uses **Winston** via `nest-winston`. Two transports:

- **Console** — always active. Colourised in dev, JSON in production.
- **Grafana Loki** — active when `LOKI_URL` env var is set. Labels: `app=wordchord`, `env=production`.

Set `LOKI_URL` to your Grafana Cloud Loki endpoint:

```
LOKI_URL=https://<userId>:<apiKey>@logs-prod-XXX.grafana.net
```

Get the URL from **Grafana Cloud → Connections → Data sources → Loki → Details**.
