---
title: Auth
description: Authentication endpoints
---

Base path: `/api/auth`

All auth endpoints are public (no token required).

---

## Register

<ParamField method="POST" path="/api/auth/register" />

**Body**

| Field | Type | Rules |
|-------|------|-------|
| `email` | string | Valid email, unique |
| `username` | string | Min 3 chars |
| `password` | string | Min 8 chars, uppercase + lowercase + digit |

**Response `201`**
```json
{
  "token": "<jwt>",
  "user": { "id": "uuid", "email": "...", "username": "...", "avatar": null }
}
```

---

## Login

<ParamField method="POST" path="/api/auth/login" />

**Body**

| Field | Type |
|-------|------|
| `email` | string |
| `password` | string |

**Response `200`** — same shape as register.

**Errors**
- `401` — invalid email or password

---

## Google Sign-in

<ParamField method="POST" path="/api/auth/google" />

**Body**

| Field | Type |
|-------|------|
| `credential` | string | Google ID token from `@react-oauth/google` |

Creates the account if it doesn't exist, otherwise signs in.

**Response `200`** — same shape as register.

---

## Forgot Password

<ParamField method="POST" path="/api/auth/forgot-password" />

**Body**

| Field | Type |
|-------|------|
| `email` | string |
| `lang` | `"en"` \| `"he"` | Email language |

Sends a reset link valid for 1 hour. Always returns `200` regardless of whether the email exists (prevents enumeration).

---

## Verify Reset Token

<ParamField method="GET" path="/api/auth/reset-password/verify" />

**Query params**

| Param | Type |
|-------|------|
| `token` | string |

**Response `200`**
```json
{ "valid": true }
```

---

## Reset Password

<ParamField method="POST" path="/api/auth/reset-password" />

**Body**

| Field | Type | Rules |
|-------|------|-------|
| `token` | string | From the reset email |
| `password` | string | Min 8 chars, uppercase + lowercase + digit |

Marks the token as used. Token can only be used once.

**Errors**
- `400` — token expired or already used
