---
title: Users
description: Profile and password management
---

Base path: `/api/user`

All endpoints require `Authorization: Bearer <token>`.

---

## Get profile

<ParamField method="GET" path="/api/user/me" />

**Response `200`**
```json
{
  "id": "uuid",
  "email": "user@example.com",
  "username": "myname",
  "avatar": "adventurer:myname"
}
```

The `avatar` field can be:
- `"style:seed"` — a DiceBear avatar (e.g. `"bottts:myname"`)
- A `data:image/...` base64 string — uploaded photo
- A full `https://` URL — external image
- `null` — defaults to `adventurer` style with the username as seed

---

## Update profile

<ParamField method="PUT" path="/api/user/profile" />

**Body** (all fields optional)

| Field | Type |
|-------|------|
| `username` | string |
| `email` | string |
| `avatar` | string \| null |

**Errors**
- `409` — email already taken

**Response `200`** — updated profile object.

---

## Change password

<ParamField method="PUT" path="/api/user/password" />

**Body**

| Field | Type | Rules |
|-------|------|-------|
| `currentPassword` | string | Must match existing hash |
| `newPassword` | string | Min 8 chars, uppercase + lowercase + digit |

**Errors**
- `400` — account uses Google sign-in (no password to change)
- `403` — current password is incorrect

**Response `200`** `{ "success": true }`
